What Ships With DollhouseMCP

The 2.0 architecture brings the product surface into one coherent system: reusable elements, a local portfolio, a public collection, and a structured query layer for MCP operations.

Elements

Six reusable element types

Personas, skills, templates, agents, memories, and ensembles give you a composable model for persistent AI behavior.

Portfolio

Local-first portfolio workflow

Your Dollhouse elements live in a readable local portfolio you can edit, back up, sync, and activate across sessions.

MCP-AQL

Semantic CRUDE endpoints

MCP-AQL groups operations into Create, Read, Update, Delete, and Execute so the AI can discover capabilities with introspection instead of juggling dozens of flat tools.

Safety

Gatekeeper and approval-aware execution

Server-side permission checks, approval flows, and execution telemetry keep agent behavior auditable even when the client is highly permissive.

How the platform works

Create or edit a Dollhouse element in natural language. Store it in your local portfolio. Activate it when you need it. Combine it with other elements in ensembles. Install community-built elements from the Collection when you want a faster starting point. 

npm install @dollhousemcp/mcp-server
npx @dollhousemcp/mcp-server@latest

Element model

Personas shape behavior, skills add capabilities, templates structure output, agents run goals, memories persist context, and ensembles coordinate multiple elements together.

Local portfolio

DollhouseMCP works from a local portfolio, so the things you build stay readable, portable, and under your control.

Collection and sharing

The Dollhouse Collection is the public browse-and-install path for community elements, making it easier to discover working building blocks instead of reinventing them.

Agent execution

Runtime execution stays inside an approval-aware loop with server-side checks, step recording, and a clear path for human input when autonomy should pause.

Why 2.0 matters

  • MCP-AQL

    Five semantic endpoints reduce tool sprawl, improve discoverability, and let the server describe itself at runtime through introspection.

  • Gatekeeper

    Permissions are enforced server-side, so active Dollhouse elements can shape what the AI can do, not just how it sounds.

  • Portfolio and backups

    Elements stay local-first with backup, sync, and collection workflows instead of disappearing into one-off chat history.

  • Composable execution

    Agents, memories, and ensembles turn isolated prompts into reusable workflows that can be inspected, refined, and run again.
